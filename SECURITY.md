# Security Policy

## Supported version

Security fixes are applied to the `main` branch while LevelFit is in MVP development.

## Reporting a vulnerability

Do not open a public issue with exploit details, credentials, personal data or health data.

Use GitHub Private Vulnerability Reporting for this repository. Include the affected endpoint or component, reproduction steps, impact and a safe proof of concept when possible.

Never include real user health data, progress photos, access tokens, refresh tokens, passwords or production secrets in a report.

## Response goals

- Acknowledge a valid report within 3 business days.
- Triage severity and affected data within 5 business days.
- Revoke exposed secrets immediately.
- Prioritize critical fixes before feature work.

These are response goals, not a bug bounty commitment.
